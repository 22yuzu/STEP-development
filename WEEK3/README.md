## メンターさんへの質問・考えたこと

### 1. HW3の括弧への対応方法について

HW3で括弧に対応する方法として、思いついたのは以下のものです。

- 今回実装した方法：括弧の中身を見つけて、その中身を `evaluate_basic` で計算し、括弧全体を `NUMBER` token に置き換える方法です。
- 再帰を使う方法：Slackに書かれていた再帰下降パーサーではなく、括弧を見つけるたびに関数をもう一度呼び出して処理するような、よりシンプルな再帰の方法を考えました。
- 数字用のstackと演算子用のstackの2つを使う方法

今回は、再帰の実装にあまり慣れていないため難しく感じたのと、stackを使う方法だとhw2で書いた処理を活かしにくいと感じたため、今の実装を選びました。
この場合、どの方法を選ぶのが一番よかったのか、それぞれの方法にどのような違いがあるのかを知りたいです。

### 2. 括弧への対応方法の種類について

HW3の括弧への対応方法は、5種類以上あるとスライドに書いてありました。
1に書いたものの他は思いつかなかったのですが、括弧に対応する代表的な方法、それぞれのメリット・デメリットを知りたいです。

### 3. `evaluate_parentheses` で毎回tokensを作り直していることについて

HW3とHW4の `evaluate_parentheses` では、括弧を1つ処理するたびに `tokens` を新しく作り直しています。
この方法は自分にとって理解しやすかったのですが、、毎回tokensをコピーして作り直しているので、効率はあまり良くないのではないかと思いました。
今回の宿題ではこの実装で問題ないのか、もしより良い方法があるならどのように改善できるのかを知りたいです。

### 4. テストケースの書き方について

今回の宿題では、不正な入力は与えられないものとして実装しました。なので、テストケースも基本的には正しい入力に対するものを中心に適当に書きました。
このような場合、どのようなテストを書くのがよいのかを知りたいです。
